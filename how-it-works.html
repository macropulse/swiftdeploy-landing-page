Start Deploying

From Chat to Cloud in Minutes

Five steps. Zero knowledge of Docker, Terraform, load balancers, or SSL required.

1

Connect your AI agent

Add the SwiftDeploy MCP server to Cursor or Claude Code. One config line — that's it.

# Add to your MCP config (Cursor / Claude Code)
"swiftdeploy": {
"url": "https://mcp.swiftdeploy.ai/mcp",
"headers": { "Authorization": "Bearer <your-api-key>" }
}

Get your API key from portal.swiftdeploy.ai after signing up.

2

Initialize your project

SwiftDeploy scans your codebase, detects the stack, and creates a project in the portal. It reads your Dockerfile if present, or auto-generates one.

You: @SwiftDeploy set up my project
SwiftDeploy: Detected Next.js 14 + Prisma + Redis. Project created.
SwiftDeploy: Connect your AWS account to continue → portal.swiftdeploy.ai
3

Connect your AWS account

One-click CloudFormation stack creates a least-privilege IAM role in your AWS account. SwiftDeploy assumes this role — it never stores your credentials.

The IAM role trust policy is scoped to your unique account UUID — no other SwiftDeploy customer can assume it, even with a valid API key.

4

Review the architecture plan

SwiftDeploy generates Terraform from one of four infrastructure profiles based on your stack. You see the full resource list and estimated monthly AWS cost before anything is provisioned.

compute-ecs
VPC · ALB · ECR · ECS · Secrets Manager
full-stack-ecs (most common)
Above + RDS + ElastiCache Redis
production-hardened
Above + autoscaling + CloudWatch alarms
edge-only
VPC · ALB · Route53 · ACM only
5

Deploy

SwiftDeploy runs tofu apply, streams progress back to your agent, auto-fixes common errors, and returns a live HTTPS URL on a SwiftDeploy subdomain.

You: deploy my-app to staging
SwiftDeploy: Deploy will consume 50 credits and create AWS resources costing ~$67/mo. Confirm?
You: yes
⟳ Deploying to staging...
[1/5] Building container... ✓
[2/5] Pushing to ECR... ✓
[3/5] Creating VPC and ECS... ✓
[4/5] Provisioning database... ✓
[5/5] Configuring SSL and DNS... ✓
✓ Deployment successful!
🌐 https://my-app-staging.app.swiftdeploy.ai

After your first deploy

Promote to production

When staging looks good, promote to production with one command. SwiftDeploy blocks promotion if staging is unhealthy.

promote staging → production

Rollback instantly

Something wrong? Roll back to the previous deployment in seconds. Production rollbacks require explicit confirmation.

rollback production

Persistent memory

SwiftDeploy remembers your infrastructure across sessions. No rebuilding context every time — your agent picks up exactly where it left off.

what's the status of staging?

Automatic error recovery

When Terraform fails with a known AWS edge case, SwiftDeploy diagnoses and fixes it automatically — no manual intervention needed.

DBSnapshotAlreadyExists — deletes the blocking snapshot automatically, then retries
Secrets Manager deletion window — sets recovery window to 0 and retries destroy
Stale state lock — clears the DynamoDB lock and resumes the apply
ECR repository not empty — clears images before decommission retries

If auto-remediation exhausts retries, a recovery_hint is returned explaining exactly what to do next.
